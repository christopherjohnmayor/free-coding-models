# Fix sticky TUI header visibility

## Context

The main TUI table should keep the app/version line, search/filter line, and column labels visible in the normal table view. The current rendered screen hides the first two header rows, so the fix needs to start from the actual terminal output and then patch the renderer with minimal scope.

## Subtasks

- [x] Reproduce the missing header with agent-tui screenshots
  report: `agent-tui` daemon fails in this environment, so I reproduced through a real Codex PTY. The raw frame shows the renderer emitting more lines than the terminal height; this makes the alternate screen scroll and drops the top header rows.
- [x] Inspect table rendering, clear-screen, cursor, and scroll calculations
  report: Found the root cause in the render budget: the constants reserve 4 header + 1 footer lines, but the normal view actually renders 3 header lines and at least 3 footer lines, plus optional banners. The viewport can therefore overfill the terminal and push sticky header rows off-screen.
- [x] Patch the renderer so the full header stays visible
  report: Updated fixed-line counts, viewport calculation, footer padding, and stale-content clearing so the table no longer emits more rows than the terminal height.
- [x] Validate with agent-tui in normal and scrolled states
  report: `agent-tui` daemon failed to accept socket/TCP connections here, so validation used a real PTY. The rendered table showed the app/version line, search/filter row, column headers, model rows, and full footer in one screen.
- [x] Run project verification commands
  report: `pnpm test` passed. `pnpm start` rendered in a PTY without runtime errors and was stopped cleanly with Ctrl+C.

## Notes

- Do not release, push, or publish without explicit user approval.
- Keep the fix focused on header visibility and avoid catalog or feature refactors.
- User clarified that the collapsed footer feature should be removed. The main table must always render both header and full footer.
